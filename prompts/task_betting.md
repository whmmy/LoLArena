## 分析要求

1. 先把"所有模型对胜负/比分/总局数的共识"提炼出来（例如 5 个模型平均胜率、大家最倾向的比分），作为你判断"模型概率"的依据。
2. 对每一个**已开盘**的盘口：
   - 计算隐含概率（建议按 95% 返还率折算）。
   - 计算凯利比例 `f = (b·p − q) / b`（`b=赔率−1`，`p=模型概率`，`q=1−p`）。
   - 与模型共识概率对比，判断是否存在价值。**凯利 `f ≤ 0` 一律不算价值；`f` 极小的只算彩票注。**
   - 用简体中文写出价值分析叙述（每段包含：模型共识概率、隐含概率、凯利 `f`、是否值得下注、建议半凯利仓位（占本金%）、与其它盘口的相关性、主要风险）。
3. 特别注意：
   - **高赔率（≥5.0）的冷门注**即使 `f>0` 也容易是估计误差放大，默认只分析不进表；除非模型概率 ≥ 2× 隐含概率且有强逻辑支撑。
   - **同向暴露**：押同一支队伍的"独赢 / 让分 / 某比分"会在同一种赛果下一起赢一起输，不能当作分散风险的多注。
4. 写一段**「组合建议与风险自检」**：点明哪些候选注同向重叠、最终保留哪几个（相互独立、edge 最大）、以及——"若失败情景发生合计亏损约占本金多少 %，若成功情景发生合计回报约多少 %"。
5. 最后输出推荐表，**严格**使用下面的表头（程序会解析，请勿改动列名与顺序）：

`| 市场 | 选择 | 赔率 | 模型概率 | 隐含概率 | 价值 | 信心 | 理由 |`

- 只把价值=`有`、且通过独立性筛选的投注项列入推荐表，**最多 3 行**。
- 价值列只填 `有` 或 `无`（仅凯利 `f>0` 且 edge 显著时填 `有`）；信心列填 `高` / `中` / `低`。
- 理由建议带上凯利仓位，如 `半凯利8%，独立强edge`。
- 若全场无价值盘口，仍输出表头，并附一行注明"本场无价值盘口"。

记住：先叙述、再组合自检、后表格；推荐表最多 3 行；只输出内容本身，不要输出 markdown 围栏包裹整段。
